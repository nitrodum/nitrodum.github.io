Step 2 — Create a new repository Click New repository Name it exactly: username.github.io (replace username with your GitHub username) Set it to Public Check "Add a README" — click Create repository This special name tells GitHub to treat it as your Pages site automatically. Step 3 — Upload your files Two ways to do this: Option A — Browser upload (easiest to start): Go to your repo → click Add file → Upload files Drag your index.html, /css, /js, /images folders in Commit directly to main Option B — Git (recommended, lets you update easily): bash git clone https://github.com/username/username.github.io # Copy your files into that folder git add . git commit -m "Initial portfolio launch" git push origin main Step 4 — Enable GitHub Pages Go to your repo → Settings → Pages Under "Source," select Deploy from a branch Branch: main / Folder: / (root) Click Save GitHub will give you a green banner with your live URL: https://username.github.io It takes 1–2 minutes to go live the first time. Every future push auto-deploys in under 60 seconds.
